	1. What is Semantic HTML? 
Tags and elements that describe what they do to the browser and programmer reading them, like h1 or section or header, as 
opposed to non-semantics like div or span.
    
	2. What is HTML used for? 
A markup language that renders on page content for the user

	3. What is an attribute and where do we put it? 
an attribute references content we want to be available to the user but not render on page, like href for <a></a> element

	4. What is the h1 tag used for? How many times should I use it on a page? 
its the highest level of headings, so also the largest and most important, as compared to h2-h6, and should only be used 
on a page once.

	5. Name two tags that have required attributes
<img> with src being the attribute, and <a><a/> with href being the attribute

	6. What do we put in the head of our HTML document? 
<!DOCTYPE html>
	<head>
		<meta charset="utf-8">
		<html lang="en">
		<link rel="stylesheet" type="text/css" href="styles.css">
	</head>
		
    7. What is an id? 
a type of selector meant for one element, like <h1 id="example"></h1>, with the css being #example {color: aqua;}

    8. What elements can I add an id to? 
h1-h6, p, ul, li, div, section

    9. How many times can I use the same id on a page? 
once

    10. What is a class?
 a selector that can be applied to multiple elements, like <p class="example"></p> with css being .example {font-size: 18px; color: aqua;}

    11. What elements can I add a class to? 
h1-h6, p, ul, li, div, section

    12. How many times can I use the same class on a page? 
multiple, not sure that there is a limit but you wouldn't hit it

    13. How do I get my link to open in a new tab?

    14. What is the alt attribute in the image tag used for?

    15. How do I reference an id?

    19. What is the difference between a class and an id? - Give me an example of when I might use each one
id targets one element whereas class targets multiple, you could use an id to change the look of just the payment button 
on a checkout page #payment-button {padding: 30px; color: green, font-weight:bold;}, versus using a class to make sure all paragraphs are the same font family .paragraphfont {font-family: comic sans MS;}
